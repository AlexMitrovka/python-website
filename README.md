# Аналіз розряду АКБ (Streamlit)

Веб-застосунок для **аналізу CSV-логів батарейного тестера**: віддана ємність (mAh), енергія (Wh), просадка напруги при підключенні навантаження, оцінка **внутрішнього опору DC**, **C-rate** та зведені оцінки якості. Є режим для **одного акумулятора** і режим **порівняння до 5 АКБ**.

**Мова інтерфейсу:** українська.

### Основні можливості

| Можливість | Опис |
|------------|------|
| **Один АКБ** | Завантаження одного CSV, таблиця метрик, графіки, оцінки 1–10, загальний бал |
| **Порівняння АКБ** | До 5 CSV, підсумкова таблиця, рейтинг, зважений бал 0–100, графіки |
| **Метрики** | mAh, Wh, середня напруга під навантаженням, ΔV (просадка), IR DC, C-rate, час тесту |
| **Оцінки** | За номіналом (mAh/Wh), порогові шкали для просадки та IR, загальний **Score** |

---

## Навіщо це потрібно

- **Оцінити реальний стан** АКБ після розряду (не лише «мА·год з лічильника», а й просідання напруги та опір).
- **Порівняти кілька батарей** у однакових умовах (той самий струм/відсічка — див. обмеження нижче).
- Дивитися **енергію (Wh)** і **середню напругу** — інколи вони краще відображають поведінку під навантаженням, ніж лише mAh.
- Отримати **підсумковий рейтинг** і текстові підказки (вердикт) для режиму одного АКБ з заданим номіналом.

## Формат CSV

Програма шукає **рядок заголовка**, що починається з **`DATE`**, і підтримує роздільник **кома**, **крапка з комою** або **табуляція**.

### Мінімально потрібні колонки

| Колонка (приклад назви) | Призначення |
|-------------------------|-------------|
| `DATE` | Час/дата точки (timestamp) |
| `VOLTAGE(V)` | Напруга, В |
| `CURRENT(A)` | Струм, А (може бути від’ємним для розряду) |
| `E_CAPACITY(mAh)` | Лічильник накопиченої ємності, мА·год |
| `E_QUANTITY(Wh)` | Лічильник накопиченої енергії, Вт·год |

Точні назви можуть трохи відрізнятися залежно від тестера; головне — щоб були **напруга, струм, ємність, енергія та час**.

Додатково корисні (якщо є в файлі): **`POWER(W)`**, колонки з **температурою** (NTC тощо).

## Єдиний стандарт імені CSV-файлу (опційно)

Це **не обов’язково** для відкриття файлу й розрахунку mAh/Wh з логу. Шаблон потрібен, якщо хочете:

- **відтворюване порівняння** — у таблиці одразу видно вагу, напруги зберігання, **IR AC** з паспорта тощо;
- **заповнити прогалини**, коли в CSV **немає** записів «до» старту розряду: програма може взяти **Ustart** з імені файлу для стартової напруги, просадки **ΔV** і узгодженого **IR DC** (якщо з логу ΔV порахувати не вдалося).

### Правила імені

| Правило | Пояснення |
|---------|-----------|
| **Латиниця** | Токени пишуться **англійськими** літерами (`W`, `Ucell`, `mOhm` тощо). |
| **Без пробілів** у назві або замініть пробіли на **`_`**. |
| **Роздільник фрагментів** — символ **`_`** (підкреслення). |
| **Префікс** | На початку може бути будь-який текст: `BrandA`, `iPhoneXs5`, дата тощо. |
| **Порядок токенів** | Після префікса токени можна ставити **в будь-якому порядку**, головне — **впізнавані** префікси з таблиці нижче. |
| **Дроби в числах** | Допускається **крапка** або **кома** в числі (кома в імені файлу перетворюється на крапку). |
| **Варіант з `_` перед числом** | Наприклад: `Ucell_4.35V` — теж розпізнається. |

### Розшифровка токенів

| Токен у імені файлу | Що означає | Де з’являється в застосунку |
|---------------------|------------|-----------------------------|
| **`W###g`** | **Вага** акумулятора, грам (наприклад `W40.7g` або `W_40.7g`) | Таблиця «Розпізнано з імені файлу», підказки на графіках |
| **`Ucell#.#V`** | **Напруга комірки** — номінал / «повний заряд», В (`Ucell4.35V`) | Там само |
| **`Ustor#.#V`** | **Напруга зберігання** (рекомендована для зберігання), В | Там само |
| **`IR###mOhm`** | **Внутрішній опір AC** до тесту, мОм — зазвичай вимір **змінним струмом** (наприклад ~1 kHz), **не** те саме, що IR з розряду | Колонка **«IR AC (мОм)»** у порівнянні |
| **`Ustart#.#V`** | **Напруга на старті** розряду (до підключення значного навантаження), В | Якщо з CSV не вийшло взяти «напругу до старту» — підставляється як стартова U; також для відновлення **ΔV** = Ustart − U після старту |

### Приклад повної назви

```text
BrandA_W45g_Ucell4.35V_Ustor3.85V_IR52mOhm_Ustart4.20V.csv
```

- `BrandA` — довільний префікс (бренд, номер зразка).
- Решта — токени з таблиці; **IR…mOhm** у застосунку трактується як **IR AC** (з паспорта/виміру до тесту).

### Контроль «повного» набору у порівнянні

Після завантаження кількох CSV у режимі **«Порівняння АКБ»** показується таблиця **«Розпізнано з імені файлу»**. Якщо знайдені **усі п’ять** значень (вага, Ucell, Ustor, IR, Ustart), у колонці **«Стандарт»** буде позначка на кшталт **OK**; якщо чогось не вистає — з’явиться підказка **доповнити ім’я файлу** потрібними токенами.

### IR AC з імені vs IR DC з тесту

- **IR AC (мОм)** — з токена **`IR…mOhm`** у назві файлу (типовий **AC**-вимір).
- **IR DC (мОм)** — з **розряду** у логу: **R = |ΔV / I|** (просадка при старті та струм), у таблиці порівняння; **не змішуйте** з IR AC — це різні величини.

## Як користуватись інтерфейсом

### Вкладка «Один АКБ»

1. У боковій панелі за потреби змініть **параметри розряду** (див. таблицю нижче).
2. Задайте **номінал АКБ** (mAh) — вручну або зі списку моделей **iPhone**; за потреби **номінальну енергію Wh** (або залиште 0 для автоматичного розрахунку).
3. Завантажте **CSV** у відповідній зоні.
4. Перегляньте **таблицю метрик**, **оцінки**, **графіки** та **вердикт**.

### Вкладка «Порівняння АКБ»

1. Завантажте **2–5 CSV** (однаковий формат бажано, порівняння коректніше при **однаковому струму** та **відсічці**).
2. За потреби вкажіть **повні назви** рядків у таблиці.
3. Оберіть **формулу рейтингу** (базова / рекомендована / компромісна — різні ваги для mAh, Wh, IR DC, ΔV).
4. Натисніть **«Порівняти»**: з’явиться **підсумкова таблиця**, **рейтинг**, **KPI**; на графіках напруги — **детальний підказувач (hover)** по кожній точці.

**Як визначається «найкращий» АКБ у порівнянні:** рядки сортуються за колонкою **«Загальний бал (0–100)»** від більшого до меншого; **1** у колонці «Рейтинг» — найвищий бал у поточній вибірці.

| Параметр | Що це | На що впливає | Типові значення |
|----------|--------|----------------|-----------------|
| **V_cutoff (напруга відсічки)** | Напруга, при якій вважається **кінець** тесту | Де обрізається інтервал розрахунку mAh/Wh | **3.2 В** для багатьох 1S Li-ion (можна змінити) |
| **I_threshold (поріг струму для старту)** | Мінімальний \|I\|, з якого починається «розряд» | Момент **старту** тесту на таймлайні | **0.2 А** — залежить від шуму в логу |
| **Вікно до старту (с)** | Скільки секунд **до** моменту старту береться для середньої напруги «до навантаження» | Оцінка **напруги до стрибка струму** та **ΔV** | **2 с** |
| **Вікно після старту (с)** | Скільки секунд **після** старту для середньої напруги «під навантаженням» | Уточнення **просадки** після підключення навантаження | **2 с** |

| **Номінал (mAh) / модель iPhone** | Очікувана «заводська» ємність для **%** і оцінок | **Оцінки mAh/Wh**, **C-rate**, загальний бал у режимі одного АКБ | Залежить від моделі телефона |
| **Номінальна енергія (Wh), 0 = з mAh×3.7 В** | Еталон Wh для **% від номіналу** по енергії | Оцінка Wh; якщо **0** — використовується **Wh_nom ≈ (mAh/1000)·3.7** | **0** для авто або своє значення |
| **Макс. точок на графік** | Обмеження кількості точок для швидкості | Графіки у порівнянні та в одному акумуляторі | **8000** |

## Метрики та як їх читати

### Ємність (mAh)

**Що це:** віддана ємність за інтервал тесту — різниця лічильника **`E_CAPACITY(mAh)`** між кінцем і початком розряду.

**Навіщо:** показує, скільки заряду «вийшло» з акумулятора до відсічки.

**Як інтерпретувати:** порівнюйте з **номіналом** (у відсотках у таблиці). Нижче ~80% номіналу часто вважають зношеним акумулятором (орієнтовно, залежить від умов).

---

### Енергія (Wh)

**Що це:** віддана енергія — різниця **`E_QUANTITY(Wh)`** на інтервалі тесту (йде з лічильника тестера).

**Чому інколи важливіше за mAh:** при різній середній напрузі під навантаженням **два тести з однаковим mAh** можуть давати **різний Wh**; для реального «наскільки батарея віддає під навантаженням» Wh іноді інформативніший.

**Зв’язок з напругою:** у середньому для оцінок порядку **Wh ≈ (середня U, В) × (Ah)**; номінальний Wh у застосунку для 1S часто задають як **(номінал mAh / 1000) × 3.7** якщо не вказано інакше.

---

### Середня напруга (V)

**Що це:** середня напруга на активному інтервалі розряду (де струм за модулем не нижче порога).

**Навіщо:** нижча середня U при тому ж струмі часто означає більші втрати/старіший акумулятор; **Wh** залежить від **U** і **I**.

---

### Просадка напруги (ΔV) на старті

**Що це:** різниця між напругою **до** підключення значного розряду (вікно до старту) і напругою **одразу після** старту (вікно після старту): **ΔV = U_до − U_після**.

**Чому велике просідання — погано:** вказує на високий **опір** і втрати; при тому ж струмі більший **ΔV** = гірша «крутість» батареї під піковим навантаженням.

---

### IR DC (мОм) — внутрішній опір у постійному струмі

**Що це:** оцінка опору з розряду: **R = |ΔV / I|** (ΔV — та сама просадка, **I** — струм після стабілізації або середній струм розряду — як у коді).

**Умови:** це **не** той самий AC-IR з імені файлу; **IR AC** у порівнянні може бути з токена в імені файлу.

**Інтерпретація:** **менший** IR DC у межах однакового тесту зазвичай **краще** (менші втрати). У режимі «один АКБ» використовується **порогова шкала** в мОм для оцінки 1–10.

---

### C-rate

**Що це:** відношення струму розряду до «номінальної ємності в А·год»:

\[
\text{C-rate} = \frac{I_{\text{сер}}}{\text{номінал (mAh)} / 1000}
\]

де \(I_{\text{сер}}\) — середній струм розряду (А), номінал — з бокової панелі.

**Навіщо:** щоб порівнювати **різні тести** справедливо, бажано однаковий **C-rate** (і ту саму відсічку). Високий C-rate «суворіший» до батареї.

---

### Загальний Score (режим «Один АКБ»)

**Що це:** зважене середнє **п’яти оцінок** (кожна в діапазоні **1–10**), якщо задано номінал і всі компоненти доступні:

\[
\text{Score} = \frac{s_{\text{mAh}} \cdot 1 + s_{\text{Wh}} \cdot 1.5 + s_{U} \cdot 1.5 + s_{\Delta V} \cdot 2 + s_{\text{IR}} \cdot 2}{8}
\]

**Що входить:** оцінка за **% mAh** і **% Wh** від номіналу, порогові оцінки за **середньою напругою**, **просадкою** та **IR** (деталі — у коді `app.py`).

**Високий бал** — ближче до **10**; **низький** — ближче до **1** (за категоріями в інтерфейсі: «Ідеал», «Дуже хороша», …).

---

### Порівняння: Score mAh / Wh / IR / ΔV та «Загальний бал (0–100)»

У таблиці порівняння кожна з метрик **нормалізується** між найкращим і найгіршим у **поточній вибірці** на шкалу **10–100**, потім зважується за обраною формулою (наприклад, 40% mAh + 25% Wh + 20% IR DC + 15% ΔV). Це **не** те саме, що оцінки від номіналу в режимі одного АКБ.

## Формули (коротко)

> **Примітка для GitHub:** у Markdown тут використано **звичайний текст** і символи **Δ**, **≈**, **×**, щоб таблиця читалась скрізь. Дубль у вигляді математики (рендер на github.com через `$…$`) — під таблицею.

| Що рахуємо | Як пояснити простими словами |
|------------|------------------------------|
| **Віддана ємність (mAh)** | Різниця лічильника **E_CAPACITY(mAh)** у кінці тесту мінус на початку (по рядках CSV між стартом і відсічкою). |
| **Віддана енергія (Wh)** | Так само — різниця **E_QUANTITY(Wh)** на тому ж інтервалі. Усе йде з лічильників тестера. |
| **Зв’язок Wh і mAh** | Точні значення — з файлу. Орієнтир: **Wh ≈ (середня U, В) × (ємність, А·год)**; часто для номіналу 1S беруть **Wh ≈ (номінал mAh / 1000) × 3.7**. |
| **IR DC** | Опір у постійному струмі: **R (Ом) = \|ΔV\| / \|I\|** (ΔV — просадка на старті, I — струм). У таблиці — **мОм** = **R × 1000**. |
| **C-rate** | **I (А) ÷ (номінал mAh ÷ 1000)** — струм відносно «однієї номінальної ампер-години». |
| **Загальний бал (один АКБ)** | Зважене середнє п’яти оцінок (1…10): мAh ×1, Wh ×1.5, U ×1.5, просадка ×2, IR ×2; сума ваг **8** → ділити на **8**. |

**Те саме в компактному вигляді (для перегляду з підтримкою математики на GitHub):**

```
ΔmAh = E_CAPACITY_кінець − E_CAPACITY_початок
ΔWh  = E_QUANTITY_кінець − E_QUANTITY_початок

R [Ом] = |ΔV| / |I|          →    IR [мОм] = R × 1000

C-rate = I / (номінал_mAh / 1000)

Score = (s_mAh + 1.5·s_Wh + 1.5·s_U + 2·s_ΔV + 2·s_IR) / 8
```

Опційно (LaTeX у доларах — на github.com може відмалюватись як формула):

$R_{\mathrm{мОм}} = \dfrac{|\Delta V|}{|I|} \times 1000$

$\mathrm{C\text{-}rate} = \dfrac{I}{\mathrm{номінал\,(mAh)}/1000}$

$\mathrm{Score} = \dfrac{s_{\mathrm{mAh}} + 1.5\,s_{\mathrm{Wh}} + 1.5\,s_U + 2\,s_{\Delta V} + 2\,s_{\mathrm{IR}}}{8}$

## Як інтерпретувати підсумковий бал

### Режим «Один АКБ» (шкала приблизно **1–10**)

| Діапазон | Орієнтовний висновок |
|----------|----------------------|
| **9–10** | Дуже добрий стан («Ідеал» / «Дуже хороша» за категоріями в застосунку) |
| **7–8.9** | Нормальний / хороший |
| **5–6.9** | Середній / знижений |
| **≤ 4.9** | Слабкий або потребує заміни (узгоджуйте з текстом вердикту в UI) |

### Режим «Порівняння» (колонка **0–100**)

Це **не абсолютна** якість, а **місце в групі** після нормалізації:

| Діапазон | Орієнтовний зміст |
|----------|-------------------|
| **90–100** | Найкращі або близькі до топа **в цій вибірці** |
| **75–89** | Добре відносно інших у порівнянні |
| **50–74** | Середній рівень у групі |
| **Нижче 50** | Слабші показники відносно інших у порівнянні |

**Важливо:** при порівнянні завжди дивіться, чи тести були **однаково жорсткі** (струм, відсічка, тривалість).
