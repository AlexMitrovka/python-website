# Аналіз розряду АКБ (Streamlit)

Веб-додаток для **аналізу CSV з батарейного тестера**: ємність, енергія, просадка напруги, оцінка внутрішнього опору (DC), оцінки якості та порівняння кількох акумуляторів.

**Мова інтерфейсу:** українська.

---

## Можливості

### Режим «Один АКБ»

- Завантаження одного CSV і повний розрахунок метрик за інтервалом розряду.
- Налаштування в боковій панелі: відсічка напруги, поріг струму для старту, вікна до/після старту, за потреби стартова напруга вручну.
- Таблиця метрик: струм, mAh, Wh, середня напруга під навантаженням, просадка, **IR DC** (R = ΔV / I), час тесту, C-rate.
- **Номінал АКБ** (mAh) — вручну або зі списку моделей **iPhone**; номінальна енергія Wh — вручну або з типової формули для 1S Li-ion.
- **Оцінки 1–10** за окремими показниками та **загальний бал** за зваженою формулою (ємність, енергія, напруга, просадка, опір).
- Графіки: напруга та струм у часі, напруга vs накопичена ємність; опційно додаткові осі.

### Режим «Порівняння АКБ»

- До **5 CSV** одночасно: підсумкова таблиця, рейтинг, KPI (найкраща ємність, найнижчий IR тощо).
- **Загальний бал (0–100)** на основі нормалізації показників у межах поточної вибірки та обраних ваг (базова / рекомендована / компромісна формула).
- Графіки порівняння; на лініях напруги — **детальний hover** по точці (час, U, I, P, накопичені mAh/Wh, dU/dt, % тесту, C-rate, локальний IR тощо).

### Ім’я файлу (опційно)

Підтримується розбір метаданих з імені файлу (вага, U зберігання, **IR AC** з імені, стартова U) для зручності в таблиці порівняння.

## Встановлення

```bash
git clone <URL-вашого-репозиторію>.git
cd python-math
pip install -r requirements.txt
```

Або віртуальне середовище:

```bash
python -m venv .venv
.venv\Scripts\activate          # Windows
# source .venv/bin/activate     # Linux / macOS
pip install -r requirements.txt
```

## Запуск

```bash
streamlit run app.py
```

У браузері відкрийте адресу, яку покаже Streamlit (зазвичай **http://localhost:8501**).

На Windows можна подвійним кліком запустити **`run_streamlit.bat`** (якщо у системі доступна команда `py`).

## Формат CSV

Очікується рядок заголовка з колонками на кшталт:

`DATE`, `VOLTAGE(V)`, `CURRENT(A)`, `E_CAPACITY(mAh)`, `E_QUANTITY(Wh)` тощо.

Програма знаходить рядок, що починається з `DATE`, і визначає роздільник полів (`,` `;` або табуляція). Час у колонці `DATE` парситься автоматично.

## Публікація в інтернеті (Streamlit Community Cloud)

1. Код на **GitHub** (у корені репозиторію мають бути **`app.py`**, **`requirements.txt`**, **`battery_compare_scoring.py`**).
2. [share.streamlit.io](https://share.streamlit.io) → увійти через GitHub → **New app** → обрати репозиторій, гілку, **Main file:** `app.py`.
3. У вікні **Advanced settings** (якщо є) спробуйте **Python 3.12** або **3.11**, якщо збірка не проходить.

### Помилка через Poetry / Replit (`repl_nix_python-website`)

У логах знову **poetry** → у репозиторії на GitHub досі є файли від **Replit**. Потрібно **один раз** прибрати старий `poetry.lock` з Git (інакше Cloud ставить зламаний lock з Flask, а не ваш `requirements.txt`).

**Варіант А (найнадійніший):** залишити лише `pip`:

1. Видаліть на GitHub **обидва** файли: `pyproject.toml` і `poetry.lock` (кнопка Delete на сторінці файлу).
2. Або локально після `git pull`:  
   `git rm -f pyproject.toml poetry.lock` → `git commit -m "Remove Poetry for Streamlit Cloud"` → `git push`
3. Переконайтесь, що в корені є **`requirements.txt`** і **`app.py`**. **Reboot** у Streamlit.

**Варіант Б:** залишити оновлений **`pyproject.toml`** з цього репозиторію (там `package-mode = false` і залежності Streamlit/Plotly), але **`poetry.lock` обов’язково видалити з Git** (`git rm poetry.lock` і push). Без видалення lock Cloud і далі читає старий lock від Replit.

Після змін — **Manage app** → **Reboot**.

### Помилка «Error installing requirements» (інше)

- **Оновіть репозиторій:** зробіть `git pull` локально, переконайтесь, що актуальний `requirements.txt` запушений (зараз там лише `streamlit` і `plotly`; `pandas` / `numpy` підтягуються разом із Streamlit — менше конфліктів версій).
- У **Manage app** відкрийте **лог** (термінал збірки) і знайдіть рядок з **червоним** текстом помилки `pip` — часто там вказано конкретний пакет.
- Перевірте, що на GitHub **немає зайвого** `requirements.txt` у підпапці й Cloud дійсно бере файл з **кореня** репо.
- Якщо після змін все одно падає — надішліть **10–20 останніх рядків логу** збірки (без секретів) у Issue або підтримку Streamlit.
